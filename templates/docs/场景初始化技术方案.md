## 2️⃣ AI Web 应用

### 技术栈

| 层级 | 技术 | 说明 |
|------|------|------|
| **框架** | Next.js 15 | App Router |
| **语言** | TypeScript | - |
| **AI SDK** | Vercel AI SDK | 流式响应、多模型支持 |
| **LLM 框架** | LangChain.js | 链式调用、工具集成 |
| **Agent** | LangGraph (可选) | 有状态工作流 |
| **样式** | Tailwind CSS | - |
| **UI** | shadcn/ui | React 组件库 |
| **验证** | Zod | Schema 验证 |

### 官方初始化方式

#### 方式 1: LangChain + Next.js 官方模板 (推荐)

```bash
# 使用 GitHub Template
# 访问 https://github.com/langchain-ai/langchain-nextjs-template
# 点击 "Use this template" → "Create a new repository"

# 或直接 clone
git clone https://github.com/langchain-ai/langchain-nextjs-template.git my-ai-app
cd my-ai-app
npm install
```

**内置 7 种模式**：
1. Basic Chat - 简单 LLM 提示
2. Structured Output - Zod 约束的 JSON 输出
3. Agents - ReAct Agent (带工具调用)
4. Retrieval (RAG) - 向量相似度搜索
5. Retrieval Agents - 带检索的工具 Agent
6. AI SDK + RSC - 流式响应 + React Server Actions
7. LangGraph - 有状态工作流

#### 方式 2: Vercel AI SDK 模板

```bash
# 访问 Vercel 模板库
# https://vercel.com/templates/ai

# 选择模板后点击 "Deploy" 或 clone
```

**热门模板**：
- Chatbot - 全功能 Next.js AI 聊天机器人
- Pinecone Starter - RAG 模式
- Morphic - AI 问答引擎

#### 方式 3: LangGraph CLI (Python 后端)

```bash
# 安装 CLI
pip install "langgraph-cli[inmem]"

# 创建项目
langgraph new my-agent-app --template new-langgraph-project

# 开发模式
cd my-agent-app
langgraph dev
```

### 官方文档

- Vercel AI SDK: https://sdk.vercel.ai/docs
- LangChain.js: https://js.langchain.com/
- LangGraph: https://langchain-ai.github.io/langgraph/

### 推荐方案

| 场景 | 推荐方案 |
|------|----------|
| 快速构建 Web Chat UI | Vercel AI SDK 模板 |
| 需要 RAG/Agent 能力 | **LangChain + Next.js 模板** (推荐) |
| 后端 Agent 工作流 | LangGraph CLI (Python) |

### 官方初始化方式

#### 方式 1: create-tauri-app (推荐)

```bash
# npm
npm create tauri-app@latest

# yarn
yarn create tauri-app

# pnpm
pnpm create tauri-app

# 其他
deno run -A npm:create-tauri-app
bun create tauri-app

# 或使用 cargo
cargo install create-tauri-app --locked
cargo create-tauri-app
```

**CLI 交互选项**：
- 项目名称
- 选择前端框架: Vanilla / Vue / Svelte / **React** / Solid / Angular / Preact / Next.js / SvelteKit
- 选择包管理器: pnpm / npm / yarn / bun / deno

#### 方式 2: tauri init (已有项目)

```bash
# 在现有前端项目中添加 Tauri
npm install -D @tauri-apps/cli
npm run tauri init
```

**CLI 选项**：
```bash
tauri init [OPTIONS]

--ci                           # 跳过交互式提示 (CI 模式)
-A, --app-name <NAME>          # 应用名称
-W, --window-title <TITLE>    # 窗口标题
-D, --frontend-dist <PATH>     # 前端资源路径
-P, --dev-url <URL>           # 开发服务器 URL
--before-dev-command <CMD>    # dev 前执行的命令
--before-build-command <CMD>  # build 前执行的命令
-f, --force                    # 强制覆盖 src-tauri 目录
```

### 官方初始化方式

#### 方式 1: oclif (Node.js 企业级 - 推荐)

```bash
# 安装 oclif CLI
npm install --global oclif

# 创建新 CLI 项目
oclif generate my-cli
```

**特点**：Heroku 出品，TypeScript 默认，插件架构，自动生成帮助文档

#### 方式 2: Commander.js (Node.js 简单)

```bash
# 安装
npm install commander

# 手写代码，无模板
```

**示例代码**：
```javascript
const { Command } = require('commander');
const program = new Command();

program
  .name('my-cli')
  .description('CLI description')
  .version('1.0.0');

program.command('hello')
  .argument('<name>', 'name to greet')
  .option('-s, --shout', 'shout the greeting')
  .action((name, options) => {
    console.log(options.shout ? `HELLO, ${name}!` : `Hello, ${name}`);
  });

program.parse();
```

#### 方式 3: clap (Rust)

```bash
# 创建 Rust 项目
cargo new my-cli

# 添加 clap 依赖
cargo add clap --features derive
```

**示例代码**：
```rust
use clap::{Parser, Subcommand};

#[derive(Parser)]
#[command(name = "mytool")]
struct Cli {
    #[command(subcommand)]
    command: Commands,
}

#[derive(Subcommand)]
enum Commands {
    Run { #[arg(short, long)] input: String },
    Build { #[arg(short, long)] release: bool },
}

fn main() {
    let cli = Cli::parse();
}
```
