# HiveLaunch 脚手架场景列表

> 基于 2025 年开发者趋势调研与市场需求分析
> 讨论日期：2025-03-08

---

## 场景概览

| Phase | 数量 | 定位 |
|-------|------|------|
| Phase 1 | 5 | 核心刚需，MVP 后优先实现 |
| Phase 2 | 4 | 高价值扩展，产品成熟后 |
| Phase 3 | 3 | 差异化机会，资源充足时 |

## Phase 1: 核心刚需 (5个)

### 1.1 📱 Expo 移动 APP

| 属性 | 内容 |
|------|------|
| **场景 ID** | `expo-app` |
| **技术栈** | Expo SDK 54 + React Native + TypeScript + NativeWind + Zustand + TanStack Query |
| **市场需求** | ⭐⭐⭐⭐⭐ 跨平台移动开发首选 |
| **AI 契合度** | 高 - AI 能大幅加速 UI 开发 |
| **现状** | ✅ 已有模板 (`react-native-template`) |
| **目标用户** | 个人开发者、创业团队、需要快速验证移动想法 |

**典型用例**：
- 社交类 APP
- 工具类 APP
- 内容消费类 APP
- O2O 服务类 APP

---

### 1.2 🤖 AI Web 应用

| 属性 | 内容 |
|------|------|
| **场景 ID** | `ai-web-app` |
| **技术栈** | Next.js 15 + React 19 + LangChain/LangGraph + OpenAI/Claude + RAG + Vector DB |
| **市场需求** | ⭐⭐⭐⭐⭐ 2024-2025 最热赛道 |
| **AI 契合度** | 极高 - AI 开发 AI 应用，自举 |
| **目标用户** | 所有想快速构建 AI 应用的开发者 |

**典型用例**：
- AI Chat / 对话机器人
- AI 写作助手
- AI 代码助手
- RAG 知识库问答
- AI Agent 应用

**市场数据**：
- LangChain Next.js 官方模板
- Vercel 专门推出 AI 模板
- GitHub Trending: AI Agent 项目 (Suna 6.5K⭐) 最火

---

### 1.3 🌐 SaaS / Dashboard

| 属性 | 内容 |
|------|------|
| **场景 ID** | `saas-dashboard` |
| **技术栈** | Next.js 15 + React 19 + shadcn/ui + Tailwind CSS + TypeScript + Prisma |
| **市场需求** | ⭐⭐⭐⭐⭐ 企业开发 80% 的需求 |
| **AI 契合度** | 极高 - CRUD + 表单 + 图表，AI 最擅长 |
| **目标用户** | 企业内部开发、SaaS 创业、后台管理系统 |

**典型用例**：
- 管理后台
- 数据看板
- SaaS 产品
- CRM / ERP 系统
- 运营工具

**市场数据**：
- create-next-app + create-vite 合计 100万+ 周下载
- Stack Overflow: React 框架 #1

---

### 1.4 📲 小程序

| 属性 | 内容 |
|------|------|
| **场景 ID** | `mini-program` |
| **技术栈** | Taro 3 + React/Vue + TypeScript + NutUI/Taro UI |
| **市场需求** | ⭐⭐⭐⭐⭐ 中国市场刚需 |
| **AI 契合度** | 高 - 小程序开发模式标准化，AI 容易上手 |
| **目标用户** | 中国开发者、微信生态创业、多端发布需求 |

**典型用例**：
- 微信小程序
- 支付宝小程序
- 字节跳动小程序
- 多端小程序 (一套代码多端发布)

**市场数据**：
- 微信日活 4.5亿+
- 人均每天使用 9.8 个小程序
- Uni-app / Taro 是主流跨端框架

**待定**：技术栈选择
- **Taro** - React 生态，与 HiveLaunch 技术栈一致
- **Uni-app** - Vue 生态，市场占有率更高

---

### 1.5 💻 Tauri 桌面应用

| 属性 | 内容 |
|------|------|
| **场景 ID** | `tauri-desktop` |
| **技术栈** | Tauri v2 + React/Vue + TypeScript + Rust |
| **市场需求** | ⭐⭐⭐⭐ 轻量桌面方案，增长迅速 |
| **AI 契合度** | 高 - 与 HiveLaunch 技术栈完全一致 |
| **目标用户** | 需要桌面端能力的应用、 Electron 替代需求 |

**典型用例**：
- 开发者工具
- 系统工具
- 桌面客户端
- 跨平台桌面应用

**市场数据**：
- Tauri v2 年增长 35%
- 二进制大小: <10MB (vs Electron 150MB+)
- 内存占用: 30-40MB (vs Electron 100MB+)

**与 HiveLaunch 关系**：
- HiveLaunch 本身就是 Tauri 桌面应用
- 模板可以复用大量现有代码

## Phase 2: 高价值扩展 (4个)

### 2.1 ⚡ REST API 服务

| 属性 | 内容 |
|------|------|
| **场景 ID** | `rest-api` |
| **技术栈** | FastAPI (Python) / Express + Prisma + PostgreSQL + TypeScript |
| **市场需求** | ⭐⭐⭐⭐ 前后端分离架构必备 |
| **AI 契合度** | 高 - API 设计模式标准化 |

**典型用例**：
- 后端 API 服务
- 微服务
- BFF (Backend for Frontend)

**技术栈选择**：
- **FastAPI** - Python 生态增长最快
- **Express/NestJS** - Node.js 生态
- **Hono** - 新兴轻量框架

---

### 2.2 📦 全栈 Monorepo

| 属性 | 内容 |
|------|------|
| **场景 ID** | `fullstack-monorepo` |
| **技术栈** | Turborepo + Next.js + Expo + tRPC + Prisma |
| **市场需求** | ⭐⭐⭐⭐ 大型项目标准架构 |
| **AI 契合度** | 高 - AI 能处理多包依赖关系 |

**典型用例**：
- 全栈产品 (Web + Mobile + API)
- 多应用共享代码
- 企业级项目

**市场数据**：
- Turborepo 采用率持续增长
- Vercel、Expo 都使用 monorepo 模式
